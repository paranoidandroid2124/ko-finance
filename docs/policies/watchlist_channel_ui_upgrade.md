# 워치리스트 채널 UI 업그레이드

## 목표
- Slack / Email 채널을 한 화면에서 비교하고, 채널별 타겟과 템플릿을 빠르게 입력할 수 있도록 한다.
- 채널별 기본 정책(필수 필드, 허용 대상, 미리보기)을 드러내어 운영팀이 "어디까지 자동화되었는가"를 명확히 이해하게 한다.

## 핵심 UX 흐름
1. **채널 선택 화면**
   - Slack / Email 카드에서 원하는 채널을 추가하고, 각 카드에 상태/최근 발송 로그/기본 플랜 제한을 표시한다.
   - 노출 필드: 채널 유형, 라벨, 기본 수신자, 추가 수신자.
2. **메시지 구성**
   - Rich Text 편집기로 본문/헤더/CTA를 작성한다(볼드/이탤릭/리스트/코드 블럭 지원).
   - 템플릿 변수 목록과 미리보기 토글을 같은 패널에 배치해 문구 검증을 빠르게 한다.
3. **검증 및 저장**
   - 채널 설정(필수 항목, 허용 도메인, 토큰 잔량 등)을 최종 확인한 뒤 저장한다.
   - 저장 버튼 옆에 최근 발송 시나리오, 실패 로그 바로가기를 둔다.
4. **즉시 발송 / 테스트**
   - 같은 화면에서 테스트 발송 / 실제 발송을 선택할 수 있도록 하고, 최근 대상/즐겨찾기 목록을 자동 완성으로 노출한다.

## 기타 고려사항
- Admin Ops 전용 경로이지만, 향후 고객-facing 워크플로와 컴포넌트 재사용을 염두에 둔다.
- Toast UI Editor 등의 레거시 의존성은 제거하고, 공통 WYSIWYG 래퍼를 도입한다.
- Step 3에서 필요한 모든 검증이 완료될 때까지 저장을 허용하지 않고, 경고 문구를 명확한 한국어로 표준화한다.
